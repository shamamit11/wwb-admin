## Implementation Steps

1. Refined the authenticated admin shell into a reusable two-region layout with persistent desktop sidebar and compact mobile navigation.
2. Added stable layout regions for topbar search, user actions, page header content, flash messaging, and an overrideable error banner slot.
3. Extracted reusable shell components for sidebar navigation, topbar search, flash stack, and signed-in user actions.
4. Updated the guest layout so auth screens render inside a consistent publishing-first shell.
5. Added a no-build Tailwind CDN fallback in both layouts so local manual verification still works while Vite is blocked by the current Node version.
6. Moved dashboard header responsibility into the admin layout and updated tests to validate the new shell behavior.
