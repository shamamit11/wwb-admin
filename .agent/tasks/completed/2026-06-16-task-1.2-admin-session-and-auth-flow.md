## Implementation Steps

1. Added a minimal service API auth client and exception layer.
2. Added a session manager for token and current-user storage using the `widewebblog` config keys.
3. Added guest/auth middleware and route aliases.
4. Added the Livewire login screen, a forbidden screen, and a logout controller.
5. Protected the dashboard route and redirected guests to login.
6. Updated the admin layout to show the current admin and a sign-out action.
7. Added focused auth feature tests with HTTP fakes against the service API contract.
