## Implementation Steps

1. Added the reusable table structure and empty-state handling needed for CRUD index screens.
2. Added pagination, loading skeleton, dialog, drawer, dropdown, and tabs primitives as Blade-native management building blocks.
3. Added admin composites for standard filter/action toolbars, row actions, confirm flows, sidebar grouping, and domain-specific status presentation.
4. Kept the APIs small and slot-driven so later Livewire screens can compose them without duplicating layout logic.
5. Added focused render tests to verify the new shared management primitives.
